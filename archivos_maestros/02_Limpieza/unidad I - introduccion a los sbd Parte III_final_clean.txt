Un Sistema de Base de Datos es el conjunto formado por la base de datos y el

software para la manipulación de la misma por parte de los usuarios que se

sirvan de ella Un SBD está compuesto por: DATOS+HARDWARE+SOFTWARE+USUARIOS En

que es un SBD, les presento esta para iniciar Usuario Final Interfaz de

aplicación Peticiones BD Esta imagen explica lo que es un SBD. Un sistema que

sirve a un usuario o varios usuarios, a través de una interfaz creada en algún

lenguaje de programación, por medio de la cual se realiza peticiones a una BD ya

elaborada, haciendo uso de un DBMS. (software especializado encargado del manejo

de una BD) Interfaz de la aplicación (de escritorio, web… BD, almacenamiento,

procesamiento de datos Un SBD se compone de lo que se conoce como Front-End y

Back-End. El Front-End se enfoca en el usuario, en todo con lo que podemos

interactuar, mientras que el Back- End está enfocado en hacer que todo lo que

está detrás de la interfaz de la aplicación funcione correctamente. Toma los

datos, los procesa y los envía al usuario, además de encargarse de las consultas

o peticiones a la Base de Datos. La separación del sistema en front-end y back-

end es un tipo de abstracción que ayuda a mantener las diferentes partes del

sistema separadas ADMINISTRADOR DE LA BASE DE DATOS (DBA) USUARIO PROGRAMADOR

Generalmente un DBA se especializa en el DBMS con el que trabaja la

organización, y su paga también depende del DBMS a manejar. No es lo mismo ser

un DBA de una BD creada en Oracle que ser un DBA de una BD creada en Access. La

arquitectura o Modelo ANSI-SPARC, (American National Standards Institute,

Standards Planning And Requirements Committee), es un estándar de diseño

abstracto para un sistema de bases de datos, fue propuesto por primera vez en el

año de 1975 por la empresa ANSI. El objetivo de esta arquitectura, también

llamada Arquitectura de tres niveles, es separar la vista de los usuarios Esta

arquitectura un enfoque de diseño para bases de datos que divide las vistas de

datos en tres capas: lo que ven los usuarios, la estructura lógica y cómo se

almacenan los datos Bajo este modelo de niveles, cada usuario debe ser capaz de

acceder a los datos que le interesan, teniendo una vista personalizada de dichos

datos. Salvo ciertas excepciones, los usuarios no deberían tener acceso a toda

la data set. Las vistas de datos deben ser independientes: los cambios en una

vista no deben afectar a las demás. Para los usuarios finales debe ser

transparentes los detalles de almacenamiento y funcionamiento del sistema. Es el

más cercano al almacenamiento físico, es decir,, describe la organización física

Algunos expertos señalan que al modelo de capas debe agregarse un subnivel

dentro del nivel lógico, se trata del nivel conceptual. Bajo este nuevo esquema,

se separa el modelo lógico de datos y la conceptualización de la realidad se

trabaja en el esquema conceptual. € Esquema Conceptual: visión desde un punto de

vista organizativo, independiente del SGBD (también conocido como DBMS) que se

utilice, e incluso de la utilización o no de sistemas de bases de datos. En este

esquema se describe la información de la organización (objetos y relaciones)

desde un punto de vista no informático. (diagrama de clases) € Esquema Lógico:

visión expresada en términos de un Sistema Gestor de Base de Datos concreto, o

mejor dicho, de un modelo de datos soportado por un SGBD. En este esquema lógico

se representan las entidades u objetos y relaciones de acuerdo a las

características de dicho modelo, sin entrar todavía en detalles de
