INTRODUCCION AL DATA WAREHOUSE . Definición: Un almacén de datos o Data

Warehouse (DWH) es un sistema centralizado que recopila, integra y almacena

grandes volúmenes de datos, provenientes de diversas fuentes, estos datos están

organizados para facilitar el análisis y la toma de decisiones. Los sistemas DWH

están diseñados específicamente para el análisis de datos, no para el registro

diario, ya que se enfoca en el análisis de la información en vez de su captura.

Imaginemos que una empresa tiene datos en Excel, en una base de datos SQL y en

un software de marketing. El DWH centraliza todo eso en un solo lugar y

normalmente es medido en gigabytes o terabytes de datos. Es importante que la

empresa cuente con un único Data Warehouse. Así, los miembros de la organización

podrán acceder a una misma fuente de información organizada según convenciones

determinadas por el management. . Diferencia entre el paradigma relacional y el

tablas, muy . Arquitectura de un DWH Un Data Warehouse no es solo una base de

datos grande; es un ecosistema. Su arquitectura se divide generalmente en capas:

. Capa de Origen: Donde viven sus bases de datos relacionales (MySQL,

PostgreSQL), archivos Excel o logs de servidores. . Área de Staging (Etapa

intermedia): ). Aquí no se consultan datos para reportes Es un lugar temporal

donde los datos "aterrizan" para ser limpiados (proceso ETL). Para que los datos

lleguen al Data Warehouse, pasan por un "túnel" llamado ETL: a. Extract

(Extraer): Sacamos los datos de las fuentes originales (sus bases de datos

relacionales). Pág. b. Transform (Transformar): Limpiamos los datos. Por

ejemplo: si en una base de datos el sexo es "M/F" y en otra es "/", lo

estandarizamos. c. Load (Cargar): Metemos los datos limpios en el DWH. . Capa de

Almacenamiento (DWH): Donde los datos ya están transformados y organizados en

modelos de estrella. . Data Marts: Son "sub-almacenes" específicos. Por ejemplo,

un Data Mart solo para Finanzas y otro solo para Recursos Humanos. Ayudan a que

cada departamento encuentre lo suyo más rápido. . ¿Cómo se organiza la

información? (Modelado Multidimensional) En lugar de pensar en muchas tablas

conectadas por llaves foráneas para evitar redundancia, en DWH se usa el Modelo

en Estrella. Componentes del Modelo: • • Tablas de Hechos (Fact Tables): Es la

tabla central. Contiene los eventos cuantitativos (lo que se puede medir).

Ejemplo: precio_vendido, cantidad_vendida. Tablas de Dimensiones: Son las tablas

que rodean a la de hechos. Responden al "quién, dónde, cuándo y qué". Ejemplo:

Dim_Producto, Dim_Tiempo, Dim_Cliente, Dim_Sucursal. DimSucursal (Dónde)

DimCliente (Quíén) Ventas DimProducto (Qué) DimTiempo (Cuando) Ejemplo: Si se

desea saber "Cuántos juguetes Mattel se vendieron en Navidad en la sucursal

Norte", la venta es el hecho, y el juguete Mattel (qué), la fecha (cuando) y la

sucursal (dónde)son las dimensiones. . Data Warehouse vs. Data Mining Es común

confundirlos, pero son dos términos diferentes • Data Warehouse: Es el

contenedor. Es donde se guarda la información de forma ordenada. • Data Mining

(Minería de Datos): Es el proceso de búsqueda. Es entrar a ese almacén con

algoritmos matemáticos para encontrar patrones que no son obvios a simple vista

(ej. descubrir que la gente que compra pañales los viernes también suele comprar

cerveza). El DWH proporciona los datos limpios; el Data Mining extrae el

conocimiento "oculto". Pág. Ejemplo de un Dashboard en Power BI . ¿Para qué

sirve todo esto? (Business Intelligence) Un Data Warehouse no es para que el

usuario final escriba INSERT. Es para que herramientas de visualización (como

Power BI o Tableau) se conecten y generen gráficos que ayuden a responder: • • •

¿Cuál es el producto más vendido en los últimos años? ¿En qué mes perdemos más

dinero? ¿Qué clientes dejaron de comprarnos este semestre? . Ventajas y

Desventajas del Data Warehouse Algunas ventajas y desventajas se presentan en el

siguiente cuadro Ventajas Desventajas Facilita el acceso y por ende análisis de

grandes volúmenes de datos Costo elevado: Implementar un DWH es caro (software y

hardware). Mejor la toma de decisiones basadas en datos históricos. Complejidad:

Requiere infraestructura y personal muy especializado. Optimiza mediante

estructuras organizadas consultas complejas Rigidez: Si el negocio cambia mucho,

el modelo debe rediseñarse . El Analista de Datos y su papel dentro del Data

Warehouse Es el profesional encargado de recopilar, limpiar e interpretar

conjuntos de datos para responder preguntas o resolver problemas de negocio. Su

objetivo principal es convertir datos crudos en información accionable (consejos

reales para los jefes). Pág. El Analista es el "usuario VIP" del Data Warehouse.

Su rol aquí es de consumo y validación: • Consulta y Extracción: Utiliza SQL

avanzado para extraer datos de los Data Marts o del DWH central. • Aseguramiento

de Calidad: Es el primero en notar si algo anda mal. Si el DWH dice que ayer se

vendieron cafés en un Starbucks de Nueva York, el analista reporta el error en

el proceso ETL. • Creación de Dashboards: Toma los datos del DWH y los conecta a

herramientas como Power BI o Tableau para que los gerentes vean gráficos bonitos

y fáciles de entender en lugar de tablas aburridas. Su papel dentro del Data

Mining Mientras que en el DWH solo mira "qué pasó", en el Data Mining busca "por

qué pasó" y "qué pasará": • • • Identificación de Patrones: El analista aplica

técnicas estadísticas para encontrar relaciones. (Ejemplo: “Noté que los

usuarios que ven series de suspenso los martes tienen un % de probabilidad de

cancelar su suscripción el próximo mes”). Segmentación: Agrupa a los clientes en

"clusters". No todos los clientes de Netflix son iguales; el analista define

quiénes son los "maratonistas", los "casuales" y los "fans del anime".

Predicción: Utiliza modelos básicos de Machine Learning para predecir tendencias

futuras basándose en los datos históricos que extrajo del almacén. Diferencia de

roles de los distintos profesionales en el flujo de datos Para que no se

confundan, vean esta jerarquía simple: . Ingeniero de Datos: Construye el Data

Warehouse (el edificio). . Analista de Datos: Explora el edificio, limpia las

salas y explica qué hay en cada una usando Data Mining y reportes. . Científico

de Datos (Data Scientist): Usa algoritmos de inteligencia artificial muy

complejos para crear productos automáticos (como el algoritmo de recomendación

de TikTok). Habilidades Técnicas que debe tener un analista de datos (El "Saber

Hacer") • SQL Avanzado (Indispensable): Se debe dominar CTEs (Common Table

Expressions), funciones de ventana (WINDOW functions) y optimización de

consultas para bases de datos en la nube (como BigQuery o Snowflake). •

Visualización y Storytelling: Manejar Power BI o Tableau, pero con un enfoque en

narrativa. Un analista exitoso no entrega un reporte; cuenta una historia donde

el "villano" es la pérdida de dinero y el "héroe" es la solución encontrada en

los datos. • • • IA: Saber usar herramientas de IA (como ChatGPT, Copilot o

Gemini) para escribir código más rápido, limpiar datos o generar ideas de

análisis. Python/R para Automatización: No se necesita ser desarrollador, pero

sí saber lo suficiente para automatizar tareas repetitivas de limpieza de datos

que en Excel tomarían horas. Estadística Aplicada: Comprender pruebas de

hipótesis y correlaciones para no decir "las ventas subieron por suerte", sino

para demostrar que el cambio es estadísticamente significativo. Pág. Habilidades

Blandas que debe tener un analista de datos (El "Saber Ser") • • • • Contexto de

Negocio: Es la habilidad #. Se debe entender cómo gana dinero la empresa. Si no

entienden se entiende el negocio, sus datos son solo números sin alma.

Pensamiento Crítico: Cuestionar los datos. Si un reporte dice algo demasiado

bueno para ser verdad, un buen analista duda y busca el error antes de

presentarlo. Comunicación para no-técnicos: Traducir "lenguaje de base de datos"

a "lenguaje de gerente". Un gerente no entiende sobre Inner Joins, pero si

entiende sobre Retorno de Inversión (ROI). Ética de Datos: En un mundo lleno de

regulaciones, saber qué datos se pueden usar y cómo proteger la privacidad del

usuario es vital Pág.
